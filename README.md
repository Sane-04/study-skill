# 学习套件：study-tutor + code-review-keeper

一套配合使用的学习类 skill。`study-tutor` 负责"把知识教懂"，`code-review-keeper` 负责"把必敲代码记牢"。两者可单独使用，也可协同形成"学 → 归档 → 复习"的完整闭环。

## 两个 skill 概览

| skill | 一句话职责 | 触发语 | 显式调用 |
|---|---|---|---|
| study-tutor | 用数理逻辑+多举例分节教你学懂，并归档必敲代码 | 带我学 / 教我 / 讲讲 / 学习模式 / 当我导师 | `$study-tutor` |
| code-review-keeper | 按艾宾浩斯曲线管理必敲代码档案并出题考你 | 复习 / 该复习哪些了 / 考考我 / 记录这段代码 | `$code-review-keeper` |

## 协作闭环

```
学习时(study-tutor)
   └─ 讲到 🔴必敲 代码 → 自动归档到桌面《必敲代码复习.md》
                              │
想复习时(code-review-keeper) ←┘
   └─ 读档案 → 按艾宾浩斯算出到期项 → 给口令让你重建 → 据反馈更新进度
```

| 场景 | 是否可用 |
|---|---|
| 只装 study-tutor | ✅ 能教学 + 能归档（自带完整归档格式） |
| 只装 code-review-keeper | ✅ 能复习已有档案 |
| 两个都装 | ✅ 学→归档→复习无缝衔接 |

## study-tutor 核心特性

| 特性 | 说明 |
|---|---|
| 先对齐再开讲 | 首轮先确认真实疑问、当前水平、学习目标 |
| 进度表头 + 小节地图 | 每次回复都显示共几节/当前第几节/每节讲什么/状态 |
| 数理逻辑优先 | 用公式、推导、维度变化讲透"为什么"，少用生活类比 |
| 多举例子 | 数值代入 / 最小可算例 / 形状演示 / 正反例 / 对比例子 |
| 分节推进 | 一次讲一节，说"继续"才进入下一节 |
| 重点分级 | 🔑重点理解 / 📌需记住 / 💡了解即可 |
| 代码分级 | 🔴必敲 / 🟡理解即可 / ⚪了解即可 |
| 面试题收尾 | 每节末用面试八股题加深理解、暴露盲区 |
| 必敲代码归档 | 讲完🔴必敲代码后自动归档到复习档案 |

## code-review-keeper 核心机制

### 复习档案
- 位置：**用户桌面下的 `必敲代码复习.md`**（不写死绝对路径，保持可迁移；不存在则自动创建）。
- 字段：编号 / 主题 / 首学日期 / 复习进度(R0~R6) / 下次复习日 / 骨架口令 / 代码。

### 艾宾浩斯节奏

| 复习点 | R1 | R2 | R3 | R4 | R5 | R6 |
|---|---|---|---|---|---|---|
| 距首学天数 | 1 | 2 | 4 | 7 | 15 | 30 |

- 过了 → 推进到下一个 R 并重算下次复习日
- 不熟 → 重置 R1（下次复习日 = 今天 + 1）
- R6 完成 → 已毕业，不再排复习

### 三类操作

| 操作 | 触发 | 行为 |
|---|---|---|
| 归档 | 记录一段必敲代码 | 追加条目（首学=今天、进度=R0、下次复习日=明天） |
| 复习判定 | "复习/考考我" | 列出"下次复习日 ≤ 今天"的条目，逾期优先，每次最多 5 条 |
| 进度更新 | 反馈"过了/不熟" | 过了→推进；不熟→重置 R1 |

考核方式：不直接贴代码，先给"主题+骨架口令"让你从零重建/讲解，再对照点评。

## 使用示例

```
# 学习
你：带我学 self-attention
AI：确认水平 → 给小节地图 → 逐节讲 → 讲到必敲代码时归档到桌面档案
你：继续 → 进入下一节

# 复习
你：复习一下今天该复习的代码
AI：读档案 → 列出到期项 → 给口令让你默写 → 对照点评
你：第一条过了，第二条不熟
AI：第一条推进，第二条重置 R1，写回档案
```

## 可迁移性

两个 skill 内均**不写死绝对路径**，归档目标统一为"用户桌面下的 `必敲代码复习.md`"，任何用户拿去都能用。档案文件属于个人数据，不随 skill 分享。

## 维护注意

两个 skill 的"必敲代码档案字段定义"各存一份且保持一致（方案 3：自包含以避免跨 skill 断链）。**若要修改字段（如新增难度列），需在两个 SKILL.md 中同步修改。**
